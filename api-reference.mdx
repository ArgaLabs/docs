The Arga API lets you start validations, retrieve run results, and manage authentication programmatically. All endpoints are available at `https://api.argalabs.com`.

## Authentication

Every authenticated endpoint accepts a Bearer token in the `Authorization` header:

```bash
Authorization: Bearer <token>
```

Arga supports two token types:

| Token type | How to obtain | Format |
|------------|--------------|--------|
| **API key** | Run `arga login` from the [CLI](/cli-and-mcp) to create a device-scoped key. | Prefixed with `arga_sk_` |
| **JWT** | Returned after completing the GitHub OAuth flow in the web app. | Standard JWT |

Both token types use the same `Authorization: Bearer <token>` header. Endpoints below note which token types they accept.

## URL validation

### Start a URL validation

<Note>Accepts both API key and JWT authentication.</Note>

```bash
POST /validate/url
```

Start a browser validation run against a deployed URL.

**Request body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | `string` | Yes | Target URL to validate. Must be a public HTTP or HTTPS URL. |
| `prompt` | `string` | Yes | Natural-language description of what to test. |
| `credentials` | `object` | No | Login credentials for the target app. |
| `credentials.email` | `string` | No | Email address for authentication. Must be provided together with `password`. |
| `credentials.password` | `string` | No | Password for authentication. Must be provided together with `email`. |
| `runner_mode` | `string` | No | Runner to use: `visual` or `dom`. Defaults to `visual` when omitted. See [runner options](/features/validate-modes#runner-options). |

**Example request**

```bash
curl -X POST https://api.argalabs.com/validate/url \
  -H "Authorization: Bearer $ARGA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://staging.example.com",
    "prompt": "test the login flow",
    "runner_mode": "dom"
  }'
```

**Response**

```json
{
  "run_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "queued",
  "session_id": "f0e1d2c3-b4a5-6789-0abc-def123456789"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `run_id` | `string` | Unique identifier for the validation run. |
| `status` | `string` | Initial run status. Always `queued`. |
| `session_id` | `string` | Chat session associated with this run. |

**Errors**

| Status | Detail |
|--------|--------|
| `400` | `A valid public http(s) URL is required.` |
| `400` | `A prompt is required.` |
| `400` | `Email and password must both be provided.` |
| `401` | Invalid or expired token. |

## PR validation story

### Start a PR validation with story generation

<Note>Requires JWT authentication.</Note>

```bash
POST /validate/start
```

Start a validation run for a pull request and stream story generation progress via server-sent events (SSE). The response includes `X-Session-Id` and `X-Run-Id` headers.

**Request body**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `pr_url` | `string` | Yes | — | Full GitHub pull request URL. |
| `frontend_url` | `string` | No | `http://localhost:3000` | Base URL of the frontend under test. |
| `context_notes` | `string` | No | — | Additional instructions or context for the validation. |
| `session_id` | `string` | No | — | UUID of an existing chat session to attach to. A new session is created when omitted. |
| `repo` | `string` | No | — | Repository in `owner/repo` format. Auto-derived from `pr_url` when omitted. |

**Example request**

```bash
curl -N -X POST https://api.argalabs.com/validate/start \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "pr_url": "https://github.com/acme/web-app/pull/42",
    "frontend_url": "http://localhost:3000",
    "context_notes": "focus on the checkout flow changes"
  }'
```

**SSE events**

| Event type | Description |
|------------|-------------|
| `investigating` | The agent is inspecting the PR. Includes `tool` and `hop` fields. |
| `sources_found` | Relevant source files identified. Includes `sources` array. |
| `story_draft` | Generated validation story. Includes `story` object and `run_id`. |
| `story_text` | Human-readable story text. Includes `content` and `run_id`. |
| `attack_plan_draft` | Generated attack plan, if applicable. Includes `attack_plan` and `run_id`. |
| `done` | Story generation complete. Includes `run_id`. |
| `error` | An error occurred during story generation. Includes `message`. |

**Errors**

| Status | Detail |
|--------|--------|
| `400` | `GitHub integration required for validation.` |
| `400` | Invalid runtime profile for the pull request. |
| `401` | Invalid or expired token. |
| `404` | `Session not found` (when `session_id` is provided but not found) |

## Sandbox deployment

### Deploy a branch

<Note>Requires JWT authentication.</Note>

```bash
POST /validate/deploy
```

Deploy a branch to an isolated sandbox environment. Returns a server-sent event (SSE) stream. The response includes `X-Session-Id` and `X-Run-Id` headers.

**Request body**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `repo` | `string` | Yes | — | Repository in `owner/repo` format. |
| `branch` | `string` | Yes | — | Branch to deploy. |
| `twins` | `array of string` | No | `[]` | Digital twin services to include in the sandbox (e.g. `box`, `discord`, `dropbox`, `google_calendar`, `google_drive`, `notion`, `slack`, `stripe`, `unified`, `unstructured`). |

**Example request**

```bash
curl -N -X POST https://api.argalabs.com/validate/deploy \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "repo": "acme/web-app",
    "branch": "feature/checkout",
    "twins": ["slack", "stripe"]
  }'
```

**SSE events**

| Event type | Description |
|------------|-------------|
| `run_created` | Sandbox run created. Includes `run_id`. |
| `environment_ready` | Sandbox environment is provisioned and ready. |
| `error` | An error occurred during deployment. |

**Errors**

| Status | Detail |
|--------|--------|
| `400` | `A repository is required.` |
| `400` | `A branch is required.` |
| `400` | `GitHub integration required for deployment.` |
| `401` | Invalid or expired token. |

### Connect an API key integration

<Note>Requires JWT authentication. Only available for providers that use API keys: `sentry`, `grafana`, `posthog`.</Note>

```bash
POST /integrations/{provider}/connect
```

Connect a provider that uses an API key or token instead of OAuth.

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `provider` | `string` | Provider name. Must be one of: `sentry`, `grafana`, `posthog`. |

**Request body**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `token` | `string` | Yes | — | API key or access token for the provider. |
| `metadata` | `object` | No | `{}` | Additional provider-specific configuration (e.g. organization slug, project ID). |

**Example request**

```bash
curl -X POST https://api.argalabs.com/integrations/sentry/connect \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "token": "sntrys_...",
    "metadata": {"organization_slug": "my-org"}
  }'
```

**Response** (`201 Created`)

```json
{"status": "connected", "provider": "sentry"}
```

**Errors**

| Status | Detail |
|--------|--------|
| `400` | Provider uses OAuth — use `/auth/{provider}/connect` instead. |
| `400` | Invalid credentials for the provider. |
| `401` | Invalid or expired token. |

## Billing

### Create a checkout session

<Note>Requires JWT authentication.</Note>

```bash
POST /billing/stripe/checkout
```

Create a Stripe checkout session to subscribe to a paid plan. Returns a URL to redirect the user to Stripe's hosted checkout page.

**Response**

```json
{"url": "https://checkout.stripe.com/c/pay/cs_..."}
```

| Field | Type | Description |
|-------|------|-------------|
| `url` | `string` | Stripe-hosted checkout URL. Redirect the user to this URL to complete payment. |

**Errors**

| Status | Detail |
|--------|--------|
| `401` | Invalid or expired token. |
| `409` | User already has a paid subscription. |
| `502` | `Stripe checkout session did not return a URL` |
| `503` | `Stripe secret key is not configured` or `Stripe price ID is not configured` |
