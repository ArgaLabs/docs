Custom scenarios let you define reusable twin seed configurations for your validation runs. Instead of starting each run with empty twins, you can pre-populate them with realistic data — Slack channels with message history, Stripe customers with active subscriptions, GitHub repos with open PRs, and more.

## Preset scenarios

Arga ships with built-in preset scenarios — ready-to-use templates covering common use cases. Presets are available without authentication and give you a starting point for common validation patterns.

Available presets include:

| Preset | Twins | Tags |
|--------|-------|------|
| E-commerce Checkout | `stripe`, `slack` | `ecommerce`, `payments`, `functional` |
| SaaS Billing & Upgrades | `stripe`, `slack`, `notion` | `saas`, `billing`, `functional` |
| DevOps CI/CD Pipeline | `unified`, `slack`, `discord` | `devops`, `ci-cd`, `functional` |
| Customer Support Portal | `slack`, `stripe`, `notion` | `support`, `customer-service`, `functional` |
| Document Collaboration | `google_drive`, `slack`, `notion` | `collaboration`, `documents`, `functional` |
| Payment Fraud & Edge Cases | `stripe`, `slack` | `security`, `payments`, `fraud` |
| Access Control & Auth Boundaries | `unified`, `slack`, `notion` | `security`, `auth`, `access-control` |
| Project Management Sprint | `unified`, `slack`, `notion` | `project-management`, `agile`, `functional` |
| Calendar & Scheduling | `google_calendar`, `slack`, `notion` | `calendar`, `scheduling`, `functional` |
| Multi-channel Notifications | `slack`, `discord`, `notion` | `notifications`, `messaging`, `functional` |

To list all presets:

```bash
curl "https://api.argalabs.com/scenarios/presets"
```

Filter by twin or tag:

```bash
curl "https://api.argalabs.com/scenarios/presets?twin=stripe&tag=payments"
```

You can also include presets when listing your own scenarios by passing `include_presets=true`:

```bash
curl "https://api.argalabs.com/scenarios?include_presets=true" \
  -H "Authorization: Bearer <api-key>"
```

Preset scenarios have `is_preset` set to `true` in the response. You can clone a preset by creating a new scenario with the same seed configuration.

## How it works

A scenario is a saved configuration that tells Arga how to seed your digital twins before a validation run begins. You can create scenarios in three ways:

- **Preset templates** — use a built-in preset scenario as-is, or clone it as a starting point for customization
- **Natural language** — describe the data you want (e.g. "3 GitHub repos, each with 3 files and 2 open PRs") and Arga generates the twin configuration automatically
- **Explicit config** — provide the exact JSON seed configuration for each twin

When you start a validation run with a scenario attached, Arga provisions the selected twins and seeds them with the scenario's data before executing any test steps.

## Creating a scenario

### From the API

Create a scenario by providing a name and either a natural language prompt or an explicit seed configuration.

**With a prompt:**

```bash
curl -X POST https://api.argalabs.com/scenarios \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "E-commerce checkout",
    "description": "Stripe with test customers and Slack with support channel",
    "prompt": "A Stripe account with 3 customers on different plans and a Slack workspace with a #support channel containing 10 recent messages about billing issues"
  }'
```

Arga's AI generates the full seed configuration from your description.

**With explicit config:**

```bash
curl -X POST https://api.argalabs.com/scenarios \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Onboarding flow",
    "twins": ["slack", "stripe"],
    "seed_config": {
      "slack": {
        "channels": [{"name": "general", "messages": ["Welcome to the team!"]}]
      },
      "stripe": {
        "customers": [{"name": "Test User", "email": "test@example.com"}]
      }
    }
  }'
```

### Supported twins

Scenarios can seed any of the following digital twins:

| Twin | What you can seed |
|------|-------------------|
| `slack` | Workspaces, channels, messages, threads |
| `discord` | Guilds, channels, messages, members |
| `github` | Repos, pull requests, issues, branches, commits, check runs |
| `stripe` | Customers, subscriptions, products, invoices |
| `notion` | Pages, databases, blocks |
| `google_drive` | Files, folders, permissions |
| `google_calendar` | Calendars, events |
| `box` | Files, folders |
| `dropbox` | Files, folders |
| `unified` | Unified API data |
| `unstructured` | Documents, workflows |

## Managing scenarios

### List scenarios

```bash
curl https://api.argalabs.com/scenarios \
  -H "Authorization: Bearer <api-key>"
```

Filter by twin or tag:

```bash
curl "https://api.argalabs.com/scenarios?twin=slack&tag=onboarding" \
  -H "Authorization: Bearer <api-key>"
```

### Update a scenario

```bash
curl -X PUT https://api.argalabs.com/scenarios/<scenario_id> \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Updated description of the data I want"}'
```

If you update the prompt without providing a new `seed_config`, Arga regenerates the configuration from the updated prompt.

### Delete a scenario

```bash
curl -X DELETE https://api.argalabs.com/scenarios/<scenario_id> \
  -H "Authorization: Bearer <api-key>"
```

## Using scenarios in validation runs

Pass a `scenario_id` when starting a validation run to seed twins with your saved configuration. The scenario's twins are provisioned and populated before test execution begins.

You can attach a scenario to:

- **PR runs** (`/validate/pr-run`) — pass `scenario_id` and optionally `scenario_prompt` to override the scenario's default prompt.
- **Twin provisioning** (`/validate/twins/provision`) — pass `scenario_id` to pre-seed standalone twin instances.

```bash
curl -X POST https://api.argalabs.com/validate/pr-run \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "repo": "acme/web-app",
    "branch": "feature/checkout",
    "scenario_id": "<scenario_id>"
  }'
```

When you provide `scenario_id` without a `scenario_prompt`, the scenario's own prompt is used as the test prompt. If you supply both, `scenario_prompt` takes precedence.

## Tags

Organize scenarios with tags for easy filtering. Tags are free-form strings you define when creating or updating a scenario.

```json
{
  "name": "Payment edge cases",
  "tags": ["billing", "edge-cases", "stripe"]
}
```
