Arga has two top-level product surfaces:

- **Ask** helps you investigate your stack with connected context sources.
- **Validate** helps you run tests, scans, and automated checks.

Within **Validate**, Arga offers three workflows. Use this page to decide which one fits your goal and how to run it.

## Which mode should I use?

| I want to... | Use |
|---|---|
| Test a live URL with AI-planned scenarios | [URL Run](#url-run) |
| Validate a PR or branch before merging | [PR Run](#pr-run) |
| Let an autonomous agent find issues I haven't thought of | [Sandbox Run](#sandbox-run) |

## URL Run

Use **URL Run** when you have a live URL and want Arga to plan and execute a comprehensive test suite. Arga automatically detects which service twins (if any) are needed based on your application.

<Info>
  **Free plan users:** URL Run is the only workflow available on the Free plan. You must provide a test prompt describing what to validate — Arga will not auto-plan tests on the Free plan. You're also limited to 1 digital twin per run and 10 runs per month. See [Plans and limits](/plans).
</Info>

### How to use it

1. Open **Validate** and select **URL Run**.
2. Enter the URL to test.
3. Optionally describe what to focus on (e.g. `test the checkout and payment flow`).
4. Optionally select a repository for additional code context — Arga uses the repo structure to inform its test plan.
5. Provide login credentials if the app requires authentication.
6. Click **Start validation**.

### What to expect

Arga probes the URL for integration signals and uses AI planning agents to generate a comprehensive test suite. If external integrations are detected, the appropriate [digital twins](/concepts/digital-twins) are provisioned automatically. If no integrations are found, the run operates as a plain browser agent. The generated plan covers:

- Core user flows and critical paths
- Integration points with external services (tested via twins)
- Edge cases and error handling
- Red-team scenarios for security and resilience

You review and approve the plan before execution begins. Arga then runs the full test suite against your URL. When twins are provisioned, they intercept all external API calls so nothing touches production services.

## PR Run

Use **PR Run** when you have code changes to validate — either manually from a branch or automatically on every pull request.

<Note>PR Run requires a Team or Paid plan.</Note>

### Manual PR Run

1. Open **Validate** and select **PR Run**.
2. Choose the repository and branch you want to validate.
3. Select which service twins to provision alongside the sandbox deployment.
4. Arga deploys the branch to a sandboxed environment, compares it against the default branch, and builds a test plan.
5. Review the proposed plan and approve it.
6. Arga provisions the sandbox, deploys your branch, and runs the full test suite.

Arga generates user stories from the branch diff, deploys your branch to a sandbox with the selected service twins, runs the test suite, and returns results with a step-by-step timeline. The generated plan includes functional tests, integration coverage via twins, and [red-team scenarios](/features/agent-security-testing) — giving you targeted coverage of exactly what changed.

### Automatic CI validation

You can also configure PR Run to trigger automatically on every pull request or on commits to a selected branch:

1. Connect GitHub in [Integrations](/integrations).
2. Open **Validate** and select **PR Run**.
3. Choose the repository you want Arga to monitor.
4. Install the Arga GitHub App for that repository.
5. Choose a trigger mode:
   - **Pull requests**
   - **Commits to branch**
6. Configure options. **PR comments** are enabled by default. You can also add **custom instructions** that apply to every run for this repository.
7. Save the validation settings.

Arga publishes results back to GitHub as the **Arga Validation** check. You can also review recent runs from the Validate UI.

For more details on setting up CI validation, see [PR validation](/features/pr-validation).

## Sandbox Run

Use **Sandbox Run** when you want to deploy an autonomous agent into a sandbox environment with service twins to explore your app and discover issues, security gaps, and edge cases.

<Note>Sandbox Run requires a Team or Paid plan.</Note>

### How to use it

1. Open **Validate** and select **Sandbox Run**.
2. Enter a URL or choose a repository and branch to deploy.
3. Optionally describe a focus area (e.g. `focus on authentication and authorization flows`).
4. Arga deploys the agent into a sandbox with the appropriate service twins.
5. The agent autonomously explores your app, probing for edge cases and vulnerabilities.

### What to expect

The agent runs adversarial and exploratory test scenarios against your app with digital twins intercepting external calls. Findings are categorized by severity (`critical`, `high`, `medium`, `low`, `info`) and attack category. The agent covers:

- Unhandled error states and edge cases
- Prompt injection and jailbreak resistance (for AI agents)
- Race conditions and state corruption
- Unexpected behaviour under simulated failures

All three workflows include [red-teaming](/features/agent-security-testing) automatically. Sandbox Run takes it further with a dedicated autonomous agent that explores beyond the boundaries of a planned test suite.
