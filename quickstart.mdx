Get Arga running against your app in three steps — no demo required.

## Get started

<Steps>
  <Step title="Install the CLI and log in">
    ```bash
    pip install arga-cli
    arga login
    ```

    This opens a browser window for authentication. Once logged in, your API key is saved locally.
  </Step>

  <Step title="Provision twins for your project">
    From your project directory, run the wizard to set up [digital twins](/concepts/digital-twins) — safe, stateful API simulators that replace your real third-party services during testing.

    ```bash
    npx arga-wizard
    ```

    The wizard will:
    1. Ask which services your app integrates with (Stripe, Slack, Discord, etc.)
    2. Scan your `.env` and rewrite API keys to point at twins
    3. Provision ephemeral twin instances (typically under a minute)

    When it finishes, start your app normally — all API calls to selected services now route through twins.

    See the full [twins quickstart guide](/features/twins-quickstart) for details on available twins, environment variables, and session management.
  </Step>

  <Step title="Run your first validation">
    Open [app.argalabs.com/validate](https://app.argalabs.com/validate) and select **Staging Run** — it's free and gives you the full experience:

    1. Enter your staging or local URL
    2. Optionally describe what to focus on
    3. Arga auto-detects your integrations, plans a test suite, and provisions the right twins
    4. Review the plan, then approve to run

    See [Validate modes](/features/validate-modes) for all four workflows.
  </Step>
</Steps>
